# MVP 项目计划

本计划严格按照当前 `README` 的主线来执行：

`NHANES 风险提示 + 知识图谱问答 + Streamlit 网页 + 综合报告`

第一阶段不把 DFUC、热成像、传感器作为主线阻塞项。

## 1. 项目目标

做出一个能演示、能解释、能汇报的最小可运行版本。

第一版验收标准不是“模型多先进”，而是：

- 有输入
- 有处理逻辑
- 有输出
- 有解释
- 有边界说明

## 4. 开发阶段

### 阶段 1：数据准备

目标：

- 把 NHANES 主线文件清洗成一张可用于风险提示的底表

任务：

- 读取 `DEMO_C.XPT`
- 读取 `DIQ_C.XPT`
- 读取 `L10_C.XPT`
- 读取 `LEXPN_C.XPT`
- 读取 `LEXAB_C.XPT`
- 读取 `SMQ_C.XPT`
- 统一按 `SEQN` 合并
- 输出 `data/processed/nhanes_risk_base.csv`

交付物：

- 数据处理脚本
- 处理后底表
- 缺失值与字段说明

### 阶段 2：风险提示模块

目标：

- 用真实结构化变量替换纯手填演示逻辑的一部分

任务：

- 定义主线字段映射
- 确定规则评分或简单模型输入字段
- 输出低 / 中 / 高风险分层
- 输出高危因素解释和就医建议

交付物：

- 风险评分模块 V1
- 示例输入输出

### 阶段 3：知识图谱问答模块

目标：

- 把固定 FAQ 和证据来源做扎实

任务：

- 扩展知识图谱种子
- 增加常见问题模板
- 为回答附上证据来源
- 增加中医护理宣教边界提示

交付物：

- FAQ 模板列表
- 图谱种子扩展版
- 问答演示页面

### 阶段 4：综合报告模块

目标：

- 把风险提示结果和问答建议整合成一页能展示的报告

任务：

- 汇总基本信息
- 汇总风险等级与高危因素
- 汇总问答建议
- 增加免责声明和边界说明

交付物：

- 报告页面
- Markdown 报告导出

### 第 1 步

- 跑通 NHANES 数据读取与合并

### 第 2 步

- 把风险评分逻辑与真实字段接上

### 第 3 步

- 固化 FAQ 模板和证据来源

### 第 4 步

- 调整综合报告页

### 第 5 步

- 做一次完整演示串联

## 7. 本周最优先任务

如果现在就开始做，我建议按这个顺序：

1. 写 `scripts/prepare_nhanes.py`
2. 生成 `data/processed/nhanes_risk_base.csv`
3. 把风险问卷页和处理后字段对应起来
4. 固化 5 到 8 个常见问题模板
5. 优化综合报告页

## 8. 完成标志

满足以下条件，就说明 MVP 已完成：

- 页面 1 可以输入并输出风险等级
- 页面 2 可以回答固定问题并给出来源
- 页面 3 可以生成结构化报告
- 可以清楚说明数据来自哪里、怎么处理、系统能解决什么问题
- 可以清楚说明系统不能做什么
