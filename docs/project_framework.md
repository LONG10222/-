# 项目框架详述

## 1. 项目名称

- 完整题目：基于公共数据与知识图谱的糖尿病足风险提示与中医护理宣教智能体
- 备选题目：基于公共数据与知识图谱的糖尿病足风险提示辅助与中医护理宣教智能体

## 2. 项目定位

本项目面向糖尿病患者居家足部健康管理与基层风险提示辅助场景，整合公开结构化临床数据、足部图像/热成像扩展数据与指南型知识文本，构建一个可用于风险提示、知识问答、护理宣教建议和网页演示的智能体原型。

项目强调三点：

- 课程可落地：公开数据可获取，网页可运行，PPT 易展示。
- 中医结合自然：聚焦护理宣教和居家管理，不强行声称做中医疗效建模。
- 技术闭环完整：主线清晰，扩展分层，网页和报告都能串起来。

## 3. 应用场景

- 糖尿病患者居家足部健康管理
- 社区/基层慢病随访中的风险提示辅助
- 糖尿病周围神经病变高风险人群教育
- 糖尿病足风险提示与就医分流

核心痛点：

- 足部问题早期不易察觉
- 患者缺乏持续自查和风险判断能力
- 医生资源有限，无法高频覆盖居家场景
- 中医辅助护理信息分散，缺少结构化知识整合

## 4. 数据资源设计

### 4.1 数据源分层

第一层：结构化临床数据

- NHANES：人口学、糖尿病相关指标、下肢疾病、周围神经病变、ABI 等模块

第二层：图像与多模态数据

- DFUC：糖尿病足溃疡图像与分割标签，作为首选图像扩展路线
- STANDUP：足底 RGB 与热成像多模态数据，当前仅保留为扩展说明与接口预留

第三层：知识文本数据

- IDF 糖尿病足指南
- CDC 足部健康与神经病变页面
- ADA/Diabetes Care 指南和综述
- 可开放获取的相关文献

### 4.2 数据资源组织目标

项目不追求医院级真实世界大数据库，也不将 NHANES、DFUC 和热成像研究数据强行拼成一个患者级统一总表，而是形成一个适合课程项目和原型系统的：

`多源异构数据资源库 + 统一字段 schema + 分任务 benchmark + 证据图谱`

建设步骤：

1. 统一 schema：把结构化数据、图像元数据和知识证据字段纳入同一套字段。
2. 标签清洗：标准化临床变量、缺失值策略、图像标签、同义实体。
3. 图谱化组织：把疾病、症状、检查、风险因子、中医护理宣教内容和管理建议连接起来。
4. 证据链记录：为每条关系保留来源、年份、来源类型和备注。

## 5. 知识图谱设计

### 5.1 实体类型

- 疾病：糖尿病、糖尿病足、糖尿病周围神经病变、感染、溃疡
- 症状：麻木、刺痛、疼痛、感觉减退、皮肤破损
- 检查：HbA1c、单丝试验、ABI、足部图像、热成像
- 风险因子：病程长、血糖控制差、吸烟、既往溃疡史、感染史
- 中医干预：足部护理、一般性宣教、生活调摄、适用边界提示
- 管理建议：每日检查、减压、复诊、转诊、线下就医

### 5.2 关系类型

- 疾病-表现为-症状
- 疾病-需要-检查
- 风险因子-增加-风险
- 中医护理建议-适用于-场景
- 异常表现-提示-转诊需求
- 指南/文献-支持-管理建议

### 5.3 样例三元组

- 糖尿病周围神经病变 - 表现为 - 麻木
- 高 HbA1c - 增加 - 糖尿病足风险
- 足部溃疡 - 提示 - 尽快线下就医
- 中医护理宣教 - 适用于 - 居家管理教育

## 6. 智能体能力设计

### 6.1 风险评估

用户输入年龄、糖尿病病程、HbA1c、足麻/刺痛/疼痛、既往足溃疡或感染史等信息，系统输出：

当前主线任务并非直接进行糖尿病足临床确诊预测，而是基于周围神经病变相关变量、下肢血供相关指标及既往病史完成综合风险分层。

- 风险等级
- 主要风险因子
- 建议就医级别
- 中医护理宣教建议

### 6.2 图像辅助识别

用户上传足部 RGB 图像，可选热成像图，系统进行：

- 异常区域启发式提示
- 红色区域/热点区域风险提示
- 是否建议进一步线下评估

### 6.3 知识图谱问答

系统针对常见问题输出结构化回答和证据来源，例如：

- 足麻是不是高风险信号？
- 什么时候必须去医院？
- 糖尿病足每天怎么护理？
- 哪些中医护理宣教内容适合居家参考？

### 6.4 报告生成

输出一份课程演示风格的综合报告，包括：

- 基本资料摘要
- 风险分层
- 图像异常提示
- 问答摘要
- 就医与护理宣教建议

## 7. 网页原型结构

建议采用 Python + Streamlit。

页面设计：

- 项目总览：场景、数据源、系统流程
- 风险问卷：结构化录入与风险评分
- 足部图像分析：优先支持 DFUC 本地样本演示，热成像保留为扩展入口说明
- 知识图谱问答：用户自由提问或选择示例问题
- 综合报告：整合问卷、图像和问答结果

## 8. 传感器与硬件路线

基础版：

- 手机 RGB 摄像头
- 普通笔记本
- Python + Streamlit
- CPU 即可运行

升级版：

- 手机外接热成像模块
- 足底压力传感器或智能鞋垫
- 本地 GPU 或云端推理服务

## 9. 应用效果

- 实现糖尿病足相关高危因素识别与风险提示
- 辅助居家足部健康管理
- 提供结构化中医护理宣教建议
- 支持基层慢病管理场景演示
- 为课程作业提供一套完整、可运行、可汇报的原型

## 10. 开发分工建议

1 号成员：需求分析、文献整理、知识图谱设计

- 场景分析
- 指南梳理
- 图谱实体关系设计
- PPT 场景与效果页

2 号成员：数据集与建模

- NHANES 字段整理
- DFUC/STANDUP 元数据整理
- 清洗规则
- 风险模型或图像模型原型

3 号成员：智能体、网页与汇报

- Streamlit 网页
- 问答流程与报告模块
- 最终演示与答辩

## 11. 项目边界说明

- 本系统用于课程项目、健康教育和辅助管理演示
- 不替代医生诊断与治疗决策
- 图像分析原型不作为临床诊断依据
- 中医建议仅限护理宣教与生活调摄方向
- 如存在足部破损、溃疡、感染或明显红肿热痛，不建议自行进行局部按压或推拿
